Easy Bridge

Timecode Protocol Bridge – User Guide
Easy Bridge receives a timecode signal from one source (LTC audio, MTC MIDI, Art-Net, or OSC) and simultaneously re-transmits it as up to three outputs: LTC audio, MTC MIDI, and Art-Net timecode. Each output can be independently enabled, offset, and configured. The app runs as a background process and can minimise to the system tray.

Header

The top bar of the window. Contains the app name, current version number, and the Help button.

ElementDescription
EASY BRIDGEApplication title. The two words use different text colours to distinguish brand from product name.
Version labelShows the current version (e.g. v2.5.8) in small text below the title.
? buttonOpens this help file in the system default browser. The file is located in the Help/ folder next to the application executable.

Timecode Clock

The large central display showing the decoded timecode in HH:MM:SS:FF format. Updated in real time as frames arrive from the active input source.

ElementDescription
Clock displayShows the current timecode position. Stays at 00:00:00:00 when no valid signal is detected. The font is monospaced for stable digit width.
TC FPS labelDisplayed below the clock. Shows the detected or configured frame rate (e.g. TC FPS: 25). Updates as the frame rate is identified from the incoming stream.

Input Source

A collapsible section that selects which timecode protocol is used as the active input. The expand/collapse arrow on the left toggles the visibility of all input rows to save screen space.

FieldDescription
Source selectorDropdown to choose the active input protocol: LTC, MTC, ArtNet, or OSC. Only the rows relevant to the selected source are used; the others are ignored.

Switching the source immediately stops the previous listener and starts the new one. All output streams continue without interruption.

Input LTC

Available when LTC is selected as the source. LTC (Linear Timecode) is encoded as an audio signal. The decoder listens on a selected audio input device and channel.

FieldDescription
DriverSelects the audio host API. Options depend on the operating system: WASAPI / DirectSound on Windows, CoreAudio on macOS. Selecting Default (all devices) lists every device regardless of driver. Use ASIO if you need ReaRoute or other ASIO devices (requires ASIO SDK build).
Device (Input)The audio input device that carries the LTC signal. The list is filtered by the selected driver. Choose the device that is physically connected to your LTC source.
ChannelWhich channel of the input device to decode. Options: 1–8 and 1+2 (stereo mix of channels 1 and 2). Most LTC setups use a single mono channel.
Sample RateThe sample rate for the audio capture. Set to Default to use the device's current rate, or choose a specific rate (44100, 48000, 88200, 96000, 176400, 192000). Must match the device's configured rate.
Level meterHorizontal bar showing the real-time input signal strength. A healthy LTC signal should have a consistent, moderate level — not silent and not clipping. If the bar is empty, check device/channel selection and verify that the source is transmitting.
Input GainAdjusts the input gain from −24 dB to +24 dB in 0.1 dB steps. The current value is shown to the right of the slider. Double-click the slider to reset it to 0 dB. Use this to normalise weak or overly hot LTC signals before decoding.
On Windows, ASIO (e.g. ReaRoute from Reaper) gives the lowest latency. WASAPI Exclusive and ASIO are preferred for reliable LTC decode over DirectSound.

Input MTC

Available when MTC is selected as the source. MTC (MIDI Timecode) is received over a standard MIDI port.

FieldDescription
MIDI InputDropdown listing all available MIDI input ports. Select the port that is sending MTC messages. Virtual ports (e.g. from DAWs) also appear here. If no ports are listed, ensure a MIDI driver is installed.

Input ArtNet

Available when ArtNet is selected as the source. Listens for Art-Net Timecode packets over UDP on port 6454.

FieldDescription
AdapterSelects the local network interface to listen on. Choose the interface connected to your Art-Net network. ALL INTERFACES listens on all adapters simultaneously (binds to 0.0.0.0).
Listen IPDisplays or overrides the bind IP address derived from the selected adapter. Normally auto-filled from the adapter selection. For manual entry use dotted-decimal notation (e.g. 192.168.1.10). Use 0.0.0.0 for all interfaces.
Ensure UDP port 6454 is allowed by the firewall on the listening machine.

Input OSC

Available when OSC is selected as the source. Receives timecode values over UDP via Open Sound Control messages. Compatible with Reaper and other DAWs that broadcast OSC position data.

FieldDescription
AdapterSelects the local network interface to bind the OSC listener. ALL INTERFACES (0.0.0.0) or Loopback (127.0.0.1) for same-machine DAW use.
Listen IPThe IP address the OSC listener binds to. Auto-filled from the adapter. Can be overridden manually.
PortUDP port number to receive OSC messages. Default is 9000. Must match the port configured in the sending application. Accepts digits only, 1–65535.
FPSFrame rate used to interpret incoming OSC float values. Options: 24, 25, 29.97, 30. Set this to match the project frame rate in your DAW.
String addressOSC address pattern for string timecode messages (e.g. /frames/str). Expected format: HH:MM:SS:FF or HH:MM:SS.FF. Default is the Reaper string position path.
Float addressOSC address pattern for numeric position messages (e.g. /time). The float value is interpreted as seconds at the configured FPS. Default is the Reaper position float path.

Output LTC

Generates an LTC audio signal from the decoded timecode and sends it to a selected audio output device. When Thru mode is active, the raw input audio is routed directly instead.

FieldDescription
Out LTC toggleEnable or disable the LTC output stream. When disabled, no audio is sent to the output device regardless of the Thru setting.
Thru dotClickable toggle in the section header. When ON (and the Out LTC switch is also ON), the raw LTC audio captured from the input device is passed directly to the selected output — no decode / re-encode cycle. This preserves the original signal bit-for-bit and eliminates encoder latency. When OFF, the normal timecode encoder output is used. The LTC input source must be running for Thru to produce audio.
DriverSelects the audio host API for output (same options as Input Driver). ASIO is preferred for lowest latency.
Device (Output)The audio output device to carry the LTC signal. Choose the device that connects to your downstream system.
ChannelOutput channel to write the LTC signal to. Options: 1–8 and 1+2 (duplicated to both channels). Use 1+2 if your downstream device requires a stereo feed.
Sample RateOutput sample rate. Must match the device's configured rate. Mismatched rates cause pitch-shifted or garbled LTC.
OffsetInteger frame offset applied to the output timecode, range −30 to +30 frames. Not applied in Thru mode (the raw audio is not time-shifted). Use it in normal mode to compensate for downstream decoding latency.
LevelOutput level in dB, range −24 dB to +24 dB. Sets the amplitude of the generated LTC audio signal. Double-click to reset to 0 dB. Adjust to match your device's expected input level.

Output MTC

Re-transmits decoded timecode as MIDI Timecode messages to a MIDI output port.

FieldDescription
Out MTC toggleEnable or disable MTC output. When disabled, no MIDI messages are sent.
MIDI OutputDestination MIDI port for MTC messages. Select the port connected to the target device (hardware or virtual).
OffsetFrame offset applied to MTC output, range −30 to +30 frames. Compensates for MIDI routing latency.
MTC loop guard: If the selected MTC Output port is the same as the MTC Input port (and MTC is the active source), the app will automatically switch the Output to the next available MIDI port. If no other port is available, MTC Output is disabled and a warning dialog is shown. This prevents MIDI feedback loops.

Output ArtNet

Broadcasts Art-Net Timecode packets over UDP to one or more IP addresses.

FieldDescription
Out ArtNet toggleEnable or disable Art-Net output.
InterfaceLocal network interface used for outgoing Art-Net packets. Choose the adapter on the same subnet as the destination devices.
Dest IP 1Primary destination IP address. Use a broadcast address (e.g. 2.255.255.255 for Art-Net universe 0) or a specific unicast IP.
Dest IP 2–5Additional destination IPs. Click the + button to add extra addresses (up to 4 extra). Click − to remove them. Each address receives the same Art-Net Timecode packet.
OffsetFrame offset applied to Art-Net output, range −30 to +30 frames.

Status Bar

The status button spans the bottom of the parameter area and shows the current operational state in text form. Click it to open the Status Monitor window.

StateMeaning
RUNNING – 25 FPSTimecode is being received and decoded at the indicated frame rate.
STOPPEDNo valid timecode is currently detected, or a configuration error prevents the listener from starting.
STOPPED – no MIDI inputMTC source is selected but no MIDI input port has been chosen or is available.
STOPPED – select LTC inputLTC source is selected but no audio input device has been chosen.
STOPPED – invalid OSC portThe OSC port field does not contain a valid integer.
STOPPED – ASIO driver failedThe ASIO backend could not initialise. Check that a 64-bit ASIO driver (e.g. ReaRoute x64) is installed.

Status Monitor Window

Opened by clicking the status button. Shows real-time diagnostic information in a compact floating panel.

FieldDescription
Input TimecodeThe raw timecode value currently being decoded, in HH:MM:SS:FF format.
FPSThe frame rate actively detected or configured for the current input source.
LTC OutputIndicates whether LTC output is active and shows its configured frame offset.
MTC OutputIndicates whether MTC output is active and shows its configured frame offset.
ArtNet OutputIndicates whether Art-Net output is active and shows its configured frame offset.
Processing LatencyInternal processing delay displayed in milliseconds and frames. Useful for diagnosing sync drift.

Settings Menu

Opened by clicking the Settings button at the bottom right. All configuration is stored in a single JSON file with a .ebc extension.

ItemDescription
Save config…Opens a file save dialog. Writes the complete current configuration (all input, output, and UI settings) to a file you choose.
Load config…Opens a file open dialog. Restores all settings from a previously saved config file. Audio and MIDI listeners restart automatically.
Reset configResets all settings to factory defaults without saving. The currently loaded file path is cleared.
Rescan audio devicesRe-enumerates audio devices in the background. Use this after connecting or disconnecting audio hardware, or after installing a new driver. Dropdowns are refreshed when the scan completes.
Load on startupToggle (checkbox). When enabled, the last saved/loaded config file is automatically loaded when the application starts. The setting itself is persisted to an internal preferences file.
Close to trayToggle (checkbox). When enabled, closing the main window minimises the app to the system tray instead of quitting it.

Troubleshooting

ProblemWhat to check
Status stays STOPPED with LTCVerify the correct input device and channel are selected. Check the Level meter — if it is empty, no audio is reaching the device. Adjust Input Gain if the signal is too weak. Confirm sample rate matches the device.
No MIDI ports listedInstall or reinstall your MIDI driver. On macOS the built-in IAC driver can be enabled in Audio MIDI Setup. On Windows use loopMIDI for virtual ports.
Art-Net not receivedCheck firewall rules for UDP port 6454. Make sure the Listen IP matches the adapter connected to the Art-Net network.
OSC not receivedVerify the Port matches the sender. Check the adapter / Listen IP setting. If sender and receiver are on the same machine, use 127.0.0.1.
LTC output sounds distortedReduce the Output Level slider. The LTC signal should be clean; clipping causes decode failures downstream.
Sync drift between outputsUse the Offset sliders on each output to compensate for downstream latency. Check Processing Latency in the Status Monitor window.
ASIO driver failed to loadEnsure you are running the 64-bit build of Easy Bridge and that a 64-bit ASIO driver is installed (e.g. ReaRoute x64 from Reaper).
Audio devices missing after reconnectUse Settings → Rescan audio devices to refresh the device lists.
LTC Thru produces silenceEnsure the LTC Input source is selected and running (the Level meter should show a signal). Thru requires an active LTC input — it copies the same audio channel being decoded. Confirm the Out LTC switch is ON and the Thru dot is ON. Select the correct output device and channel.
MTC Output auto-switched or disabled unexpectedlyThe MTC loop guard activated because the selected Output port matched the Input port while MTC was the active source. Choose a different Output port, or use a virtual MIDI loopback (e.g. loopMIDI) to create a separate port.
